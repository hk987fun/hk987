[Win基础]-8-使用命令行工具reg操作windows注册表

引言

熟悉Windows操作系统的朋友可能都知道，Windows操作系统下的注册表相当于系统的数据库 ，部分软件将自己的配置信息都放在注册表里面，而注册表的破坏可以直接导致系统的崩溃。而熟悉注册表的朋友对"注册表编辑器（regedit.exe）"一点也不陌生，它提供了一个图形化管理注册表的操作界面，本文要介绍的是另一种管理注册表的方法"控制台注册表编辑器(reg.exe)"，它可以使用纯命令行的模式对注册表进行添加、删除、查看等操作；适用于在批处理中对注册表进行操作。

0×1.Window注册表工具reg命令解析

reg命令是Windows系统命令，如果我们的系统装在"C盘"，那么控制台注册表编辑器reg.exe就位于c:\windows\system32\目录下面，它是系统修改注册表的一个工具。（一些精简版的系统将这些程序都删除了，这样的情况你可以从其他系统中拷贝一个放置这个目录下）

要调用"控制台注册表编辑器"只需要打开一个CMD命令行，输入"reg /?"就能看到帮助文件了，如下图所示（注意，Win7以后的系统中必须用管理员身份运行CMD才能对注册表的一些项进行操作，否则会因为权限不足而报错）

在下面的示例命令中，"reg add" 是添加的意思，其他根据英文类推即可，HKCU是根键"HKEY_CURRENT_USER"的缩写，另外几个根键的缩写分别为"HKCR"、"HKLM"、"HKU"、"HKCC"，取根键的第一个单词的前两个字母和后面两个单词的前一个字母。

hkcu\hackmm就是在根键"HKEY_CURRENT_USER"下添加一个叫做hackmm的子键，如果这个子键存在，则不作任何操作。/v 后面跟需要创建的值的名称。/t 后面是值的类型，/d 后面是这个值的数据。

视频中用到的命令：

					
					#1.添加注册表项（键）：
					C:\Windows\System32> reg add hkcu\hackmm

					#2.添加注册表项和值以及类型和数据
					C:\Windows\System32> reg add hkcu\hackmm /v welcome /t REG_SZ /d "欢迎来黑客萌妹一起学习"

					#3.添加注册表项默认值以及类型和数据
					C:\Windows\System32> reg add hkcu\hackmm /ve /t REG_DWORD /d 1

					#4.查询数据表项下面的所有值
					C:\Windows\System32> reg query hkcu\hackmm

					#5.查询数据表项下面的单个指定值
					C:\Windows\System32> reg query hkcu\hackmm /v welcome

					#6.查询数据表项下面的默认值，（ve=value empty）
					C:\Windows\System32> reg query hkcu\hackmm /ve

					#7.将数据表指定项导出成文件备份
					C:\Windows\System32> reg export hkcu\hackmm e:\hackmm.reg

					#8.导出整个根项/根键成文件备份
					C:\Windows\System32> reg export hkcr e:\hkcr.reg

					#9.删除某个指定的注册表项
					C:\Windows\System32> reg delete hkcu\hackmm /f

					#10.用备份的注册表文件，恢复数据
					C:\Windows\System32> reg import e:\hackmm.reg

					#11.删除指定项下指定的值
					C:\Windows\System32> reg delete hkcu\hackmm /v welcome /f

					#12.删除指定项下的默认值
					C:\Windows\System32> reg delete hkcu\hackmm /ve /f
